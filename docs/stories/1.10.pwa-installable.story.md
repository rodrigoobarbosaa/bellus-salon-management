# Story 1.10: PWA — App Instalável no Telemóvel

**Status:** InProgress
**Epic:** EPIC-1 Bellus — Wave 1 Fundação
**Criado por:** River (@sm)
**Data:** 2026-03-28
**Depende de:** Story 1.1 (setup)

---

## Story

**As a** proprietário do salão,
**I want** instalar o Bellus como app no meu telemóvel,
**so that** eu possa aceder rapidamente sem abrir o browser.

---

## Acceptance Criteria

1. Web App Manifest (`manifest.json`) configurado com nome, ícones e cores do Bellus
2. Ícones PWA criados: 192x192, 512x512 (PNG)
3. Apple touch icon (180x180) para iOS
4. Meta tags PWA no layout: theme-color, apple-web-app-capable, manifest link
5. Service worker básico registado — cache de assets estáticos
6. App abre em modo standalone (sem barra do browser) quando instalado
7. Start URL é `/dashboard` para utilizadores autenticados
8. Splash screen com cor de fundo Bellus (#FAFAF9) e ícone
9. `npm run build` sem erros
10. Lighthouse PWA score >= 80

## Dev Notes

### Web App Manifest

```json
{
  "name": "Bellus Salon Management",
  "short_name": "Bellus",
  "description": "Sistema de gestão para salão de beleza",
  "start_url": "/dashboard",
  "display": "standalone",
  "background_color": "#FAFAF9",
  "theme_color": "#C9A96E",
  "icons": [
    { "src": "/icon-192.png", "sizes": "192x192", "type": "image/png" },
    { "src": "/icon-512.png", "sizes": "512x512", "type": "image/png" }
  ]
}
```

### Service Worker Strategy

```javascript
// Cache-first para assets estáticos
// Network-first para API calls
// Skip waiting + clients claim para updates imediatos
```

### Next.js App Router

Next.js 16 suporta `manifest.ts` nativamente em `src/app/manifest.ts` que exporta um objeto tipado. Alternativa: ficheiro estático em `public/manifest.json`.

### Ícones

Sem assets de design disponíveis — gerar ícones simples com a letra "B" em fundo bellus-gold (#C9A96E) com texto branco.

### Referências

- **Layout:** `bellus/src/app/layout.tsx`
- **Public:** `bellus/public/`
- **Config:** `bellus/next.config.ts`
