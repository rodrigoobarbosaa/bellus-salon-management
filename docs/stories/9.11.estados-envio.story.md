# Story 9.11: Gestão de Estados de Envio à AEAT

**Status:** InProgress
**Epic:** EPIC-09 Verifactu
**Criado por:** River (@sm)
**Data:** 2026-03-30
**Depende de:** Story 9.10 (API AEAT)
**Complexidade:** M (5 SP)

---

## Story

**As a** proprietário de salão,
**I want** ver o estado de cada fatura relativamente ao envio à AEAT,
**so that** saiba quais foram aceites, rejeitadas ou pendentes e possa tomar ação.

---

## Acceptance Criteria

1. Cada fatura mostra badge visual de estado: pendiente (amarelo), enviado (azul), aceptado (verde), rechazado (vermelho)
2. Faturas rejeitadas mostram o motivo do rechazo (código + descrição da AEAT)
3. Existe botão "Reenviar" para faturas com estado `rechazado` ou `error`
4. Dashboard mostra contador: X faturas pendientes de envio, Y rejeitadas
5. Alerta no dashboard quando há faturas rejeitadas não resolvidas (> 24h)
6. Existe server action `reenviarFactura(facturaId)` que reenvia o XML à AEAT
7. Histórico de tentativas de envio visível no detalhe da fatura (via `factura_envios_aeat`)
8. `npm run build` sem erros
