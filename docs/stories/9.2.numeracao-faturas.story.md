# Story 9.2: Sistema de Numeração Sequencial de Faturas

**Status:** InProgress
**Epic:** EPIC-09 Verifactu
**Criado por:** River (@sm)
**Data:** 2026-03-30
**Depende de:** Story 9.1 (schema facturas)
**Complexidade:** S (3 SP)

---

## Story

**As a** proprietário de salão,
**I want** que cada fatura receba um número sequencial único e inalterável,
**so that** a numeração cumpra os requisitos da AEAT (sem saltos, sem duplicados, série identificável).

---

## Acceptance Criteria

1. Existe uma função SQL `get_next_factura_numero(p_salao_id UUID, p_serie TEXT)` que retorna o próximo número disponível de forma atómica (race-condition safe)
2. A função usa `SELECT ... FOR UPDATE` ou `advisory lock` para garantir que dois pagamentos simultâneos não gerem o mesmo número
3. A série default é `'B'` (Bellus) mas pode ser configurada por salão na tabela `configuracoes_fiscais` (novo campo `serie_factura`)
4. Números são sequenciais sem saltos dentro da mesma série e salão (1, 2, 3...)
5. O formato final é `{SERIE}-{NUMERO_6_DIGITOS}` (ex: `B-000001`, `B-000002`)
6. Ao criar uma fatura, o número é atribuído atomicamente dentro da transaction de INSERT
7. Teste manual: criar 3 faturas consecutivas → números B-000001, B-000002, B-000003
8. Teste de concorrência: 2 faturas criadas "simultaneamente" → números distintos sem salto
9. `npm run build` sem erros

---

## Scope

**IN:**
- Função SQL de numeração atómica
- Campo `serie_factura` em `configuracoes_fiscais`
- Lógica de atribuição no INSERT

**OUT:**
- Geração da fatura completa (Story 9.3)
- Hash encadeado (Story 9.4)
