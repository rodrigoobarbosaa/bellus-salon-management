## User Story

**Como** proprietário do salão,
**Quero** fazer login e logout com email e senha,
**Para que** apenas eu possa acessar o dashboard do meu salão.

## Contexto

Autenticação via Supabase Auth. O sistema tem 3 roles: proprietário, profissional, cliente. Esta story cobre o fluxo básico de auth para o proprietário.

## Acceptance Criteria

- [x] AC-01: Proprietário consegue criar conta com email + senha
- [x] AC-02: Proprietário consegue fazer login com credenciais válidas
- [x] AC-03: Login com credenciais inválidas mostra mensagem de erro clara
- [x] AC-04: Logout redireciona para página de login
- [x] AC-05: Rotas do dashboard inacessíveis sem autenticação (middleware)
- [x] AC-06: Sessão persiste ao recarregar a página
- [ ] AC-07: Página de login em 4 idiomas (PT, ES, EN, RU) — depende da Story 1.6 (i18n)
- [x] AC-08: Loading state durante operações de auth
