## User Story

**Como** proprietário do salão,
**Quero** fazer login e logout com email e senha,
**Para que** apenas eu possa acessar o dashboard do meu salão.

## Contexto

Autenticação via Supabase Auth. O sistema tem 3 roles: proprietário, profissional, cliente. Esta story cobre o fluxo básico de auth para o proprietário.

## Acceptance Criteria

- [ ] AC-01: Proprietário consegue criar conta com email + senha
- [ ] AC-02: Proprietário consegue fazer login com credenciais válidas
- [ ] AC-03: Login com credenciais inválidas mostra mensagem de erro clara
- [ ] AC-04: Logout redireciona para página de login
- [ ] AC-05: Rotas do dashboard inacessíveis sem autenticação (middleware)
- [ ] AC-06: Sessão persiste ao recarregar a página
- [ ] AC-07: Página de login em 4 idiomas (PT, ES, EN, RU)
- [ ] AC-08: Loading state durante operações de auth

## Tarefas Técnicas

- [ ] T-01: Configurar Supabase Auth no projeto
- [ ] T-02: Criar página de login (`/login`) com formulário email + senha
- [ ] T-03: Criar página de registro (`/register`) — apenas para setup inicial
- [ ] T-04: Implementar middleware Next.js para proteger rotas `/dashboard/**`
- [ ] T-05: Implementar logout com limpeza de sessão
- [ ] T-06: Criar hook `useAuth()` para estado de autenticação
- [ ] T-07: Adicionar traduções de auth para os 4 idiomas
