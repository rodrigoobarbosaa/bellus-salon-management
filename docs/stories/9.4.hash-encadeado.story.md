# Story 9.4: Hash SHA-256 Encadeado entre Faturas

**Status:** InProgress
**Epic:** EPIC-09 Verifactu
**Criado por:** River (@sm)
**Data:** 2026-03-30
**Depende de:** Story 9.3 (server actions facturas)
**Complexidade:** L (8 SP)

---

## Story

**As a** sistema Bellus certificado,
**I want** que cada fatura gere um hash SHA-256 vinculado à fatura anterior,
**so that** a cadeia de faturas seja inalterável e verificável, cumprindo o RD 1007/2023 (integridade e trazabilidade).

---

## Acceptance Criteria

1. Existe módulo `src/lib/verifactu/hash.ts` com função `calculateFacturaHash(factura, hashAnterior)` que gera hash SHA-256
2. O input do hash inclui campos obrigatórios conforme RD 1007/2023: NIF emissor, número fatura, fecha emision, tipo factura, base imponible, IVA, total, hash da fatura anterior
3. A primeira fatura de uma série usa hash anterior = `'0'` (genesis)
4. O hash é calculado no server-side (Node.js `crypto` module) — NUNCA no client
5. O campo `hash_actual` é preenchido automaticamente no momento do INSERT (dentro da mesma transaction que o número)
6. O campo `hash_anterior` referencia o `hash_actual` da última fatura emitida para aquele salão+série
7. Existe função `verifyFacturaChain(salaoId, serie)` que verifica a integridade de toda a cadeia (recalcula hashes e compara)
8. Se `verifyFacturaChain` detectar inconsistência, retorna o ponto de quebra (número da fatura)
9. A verificação de cadeia completa para 1000 faturas executa em menos de 5 segundos
10. `npm run build` sem erros

---

## Notas Técnicas

O RD 1007/2023 especifica que o hash deve incluir:
```
hash = SHA-256(
  NIF_emisor +
  numero_factura +
  fecha_emision (YYYY-MM-DD) +
  tipo_factura (F1=normal, R1=retificativa) +
  base_imponible +
  cuota_iva +
  importe_total +
  hash_factura_anterior
)
```

O encoding é UTF-8, o resultado é hex string (64 chars).
