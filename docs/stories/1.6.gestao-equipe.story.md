# Story 1.6: Gestão de Equipe — CRUD de Profissionais

**Status:** Ready for Review
**Epic:** EPIC-1 Bellus — Wave 1 Fundação
**Criado por:** River (@sm)
**Data:** 2026-03-22
**Depende de:** Story 1.2 (schema DB — tabela `profissionais`), Story 1.4 (layout dashboard), Story 1.5 (sistema i18n)

---

## Story

**As a** proprietário do salão,
**I want** poder gerenciar minha equipe de profissionais (criar, editar, ativar/desativar e atribuir serviços),
**so that** eu tenha controle completo sobre quem trabalha no salão e quais serviços cada profissional oferece.

---

## Acceptance Criteria

1. Página `/dashboard/equipe` carrega sem erro 404, acessível pelo menu lateral
2. Grid responsivo de cards mostra todos os profissionais do salão (`sm:grid-cols-2 lg:grid-cols-3`)
3. Cada card exibe: avatar colorido (cor_agenda), nome, role badge (Proprietário/Profissional), email, telefone, contagem de serviços atribuídos
4. Botão "Novo profissional" abre dialog modal com campos: nome*, email*, telefone, role (select), cor_agenda (color picker)
5. Criar profissional insere registro na tabela `profissionais` com `user_id: crypto.randomUUID()` como placeholder
6. Botão editar abre o mesmo dialog preenchido com dados existentes; salvar atualiza o registro
7. Toggle ativo/inativo altera campo `ativo` e visual do card (opacity reduzida + badge "INATIVO")
8. Busca por nome, email ou telefone filtra a lista em tempo real
9. Filtros de status: Todos / Ativos / Inativos
10. Botão "Gerenciar serviços" abre dialog com lista de serviços ativos + checkboxes + campo de preço override
11. Salvar atribuição de serviços faz delete+insert na tabela `servicos_profissionais`
12. Contagem de serviços no card atualiza após salvar atribuições
13. Todas as strings da UI traduzidas em 4 idiomas (PT, ES, EN, RU) via seção `team` nos locales
14. Apenas proprietários veem botões de criar, editar, toggle e gerenciar serviços

### Decisão: user_id para novos profissionais

O campo `user_id` é obrigatório (non-nullable no Insert type). Para profissionais criados pela equipe page (sem conta auth), usar `crypto.randomUUID()` como placeholder. Futuramente, um fluxo de convite atualizará para o user_id real.

### Padrão de Queries Supabase

Usar `from("tabela" as string)` para bypass do type-checking rígido, e cast manual dos resultados:
```typescript
const { data } = await supabase
  .from("profissionais" as string)
  .select("*")
  .order("nome");

// Cast manual
const profissionais = (data as Array<{ id: string; nome: string; ... }>) ?? [];
```
