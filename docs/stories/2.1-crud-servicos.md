# Story 2.1 — CRUD de Serviços

**Epic:** EPIC-02 Agenda & Serviços
**Status:** Ready for Dev
**Prioridade:** MUST HAVE
**Depende de:** EPIC-01 (completo)
**PRD Refs:** FR-03.1, FR-03.2, FR-03.3, FR-03.4, FR-03.5

---

## Story

Como **proprietário do salão**, quero **gerenciar o catálogo de serviços** (criar, editar, ativar/desativar), para que os profissionais possam ser associados aos serviços e os clientes vejam as opções disponíveis.

---

## Acceptance Criteria

- [ ] AC-01: Proprietário pode ver lista de serviços do salão com nome, preço, duração, categoria
- [ ] AC-02: Proprietário pode criar novo serviço com: nome, descrição, duração (minutos), preço base, categoria, intervalo de retorno
- [ ] AC-03: Proprietário pode editar serviço existente
- [ ] AC-04: Proprietário pode ativar/desativar serviço (soft delete)
- [ ] AC-05: Proprietário pode associar profissionais a serviços com preço override opcional
- [ ] AC-06: Profissional com role "profissional" pode visualizar serviços mas NÃO editar
- [ ] AC-07: Validação: nome obrigatório, duração > 0, preço >= 0
- [ ] AC-08: Interface responsiva (mobile-first)

---

## Dev Notes

### DB Schema (já existe)
- `servicos` — tabela principal (migration 003)
- `servicos_profissionais` — relação N:N com preço override (migration 003)
- `salao_id` adicionado em migration 009 (multi-tenant)
- RLS multi-tenant já configurado

### Categorias (ENUM existente)
`corte | coloracao | mechas | tratamento | outro`

### Componentes a criar
1. **Página `/dashboard/servicos`** — lista de serviços com filtro por categoria
2. **Modal/drawer de criação** — formulário com validação
3. **Modal/drawer de edição** — mesma estrutura, pré-populado
4. **Server Actions** — `createServico`, `updateServico`, `toggleServicoAtivo`
5. **Associação profissional** — selecionar quais profissionais fazem o serviço + preço override

### Padrões a seguir
- Server Components para lista (SSR)
- Server Actions para mutações
- `createClient()` (não service client — RLS filtra por salao_id)
- Componentes UI: Card, Button, Input, Select do projeto existente
- i18n: textos em espanhol (interface principal)
