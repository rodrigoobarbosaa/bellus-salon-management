## User Story

**Como** proprietário do salão,
**Quero** usar o sistema no meu idioma preferido (PT, ES, EN ou RU),
**Para que** a plataforma seja acessível para qualquer profissional em Valencia.

## Contexto

Valencia tem comunidade diversa. O sistema deve funcionar em 4 idiomas desde o início. Usar next-intl (já no projeto). Esta story configura a infraestrutura i18n — as traduções específicas de cada módulo serão adicionadas nas stories subsequentes.

## Acceptance Criteria

- [ ] AC-01: Sistema detecta idioma do browser automaticamente
- [ ] AC-02: Seletor de idioma visível no header do dashboard
- [ ] AC-03: Idioma selecionado persiste entre sessões (salvo no perfil do usuário)
- [ ] AC-04: Conteúdo base do dashboard traduzido em PT, ES, EN, RU
- [ ] AC-05: Datas e números formatados conforme locale do idioma
- [ ] AC-06: Fallback para espanhol se idioma não suportado

## Tarefas Técnicas

- [ ] T-01: Configurar next-intl com 4 locales (pt, es, en, ru)
- [ ] T-02: Criar estrutura de arquivos de tradução `src/locales/{locale}/`
- [ ] T-03: Traduções do layout base: navegação, dashboard, configurações
- [ ] T-04: Componente `<LanguageSwitcher>` no header
- [ ] T-05: Salvar preferência de idioma no perfil do usuário (Supabase)
- [ ] T-06: Configurar formatação de datas com `Intl.DateTimeFormat`
