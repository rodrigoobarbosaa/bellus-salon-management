# Story 1.3: Autenticação Multi-Profissional

**Status:** Ready
**Epic:** EPIC-1 Bellus — Wave 1 Fundação
**Criado por:** River (@sm)
**Data:** 2026-02-25
**Depende de:** Story 1.1 (projeto configurado), Story 1.2 (schema criado)

## Story

**As a** profissional do salão Bellus,
**I want** poder fazer login de forma segura com email e senha,
**so that** eu acesse apenas minha agenda e dados, com o proprietário tendo acesso completo a todos os dados do salão.

---

## Acceptance Criteria

1. Página de login (`/login`) com formulário email + senha, validação e feedback de erros
2. Login funcional via Supabase Auth (email/password)
3. Após login, redirecionar para dashboard `/dashboard`
4. Logout funcional — limpa sessão e redireciona para `/login`
5. Middleware Next.js protege todas as rotas `/dashboard/**` — redireciona não-autenticados para `/login`
6. Sessão persistida via cookies (Supabase SSR) — refresh automático sem re-login
7. Role `proprietario` tem acesso ao menu completo (clientes, fiscal, todos os profissionais)
8. Role `profissional` vê apenas sua própria agenda e seus atendimentos
9. Tela de "Esqueci minha senha" — envia email de reset via Supabase Auth
10. Proteção contra brute force: Supabase Auth rate limiting ativo
11. Página 404 personalizada para rotas não encontradas

## Dev Notes

### Supabase Auth com Next.js SSR (App Router)

O Bellus usa `@supabase/ssr` para compatibilidade com Next.js App Router:

```typescript
// src/lib/supabase/server.ts
import { createServerClient } from '@supabase/ssr'
import { cookies } from 'next/headers'

export function createClient() {
  const cookieStore = cookies()
  return createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        getAll() { return cookieStore.getAll() },
        setAll(cookiesToSet) {
          cookiesToSet.forEach(({ name, value, options }) =>
            cookieStore.set(name, value, options)
          )
        },
      },
    }
  )
}
```

### Middleware de Proteção de Rotas

```typescript
// src/middleware.ts
import { createServerClient } from '@supabase/ssr'
import { NextResponse, type NextRequest } from 'next/server'

export async function middleware(request: NextRequest) {
  // Criar response mutável para atualizar cookies de sessão
  let supabaseResponse = NextResponse.next({ request })

  const supabase = createServerClient(...)
  const { data: { user } } = await supabase.auth.getUser()

  // Proteger rotas /dashboard
  if (!user && request.nextUrl.pathname.startsWith('/dashboard')) {
    return NextResponse.redirect(new URL('/login', request.url))
  }

  return supabaseResponse
}

export const config = {
  matcher: ['/dashboard/:path*', '/login', '/'],
}
```

### Estrutura de Roles

A tabela `profissionais` tem o campo `role`:
- `proprietario` → acesso total ao sistema
- `profissional` → acesso apenas à própria agenda e atendimentos

O role é carregado após login consultando a tabela `profissionais` via `user.id`.

### Segurança — OWASP Considerations

- **Credenciais:** Nunca logar senhas. Supabase Auth cuida do hash bcrypt
- **Sessões:** Tokens JWT com expiração curta + refresh token rotativo
- **CSRF:** Next.js + Supabase SSR protegem contra CSRF automaticamente
- **Rate limiting:** Supabase Auth limita tentativas de login por IP
