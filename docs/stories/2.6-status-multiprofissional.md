## Story

Como **proprietário**, quero **alterar o status dos agendamentos** (pendente → confirmado → concluído) e **visualizar todos os profissionais simultaneamente** no calendário, para ter controle total da operação do salão.

---

## Acceptance Criteria

- [ ] AC-01: Agendamento tem indicador visual de status (cor/ícone por status)
- [ ] AC-02: Proprietário pode alterar status: pendente → confirmado → concluído
- [ ] AC-03: Cores de status: pendente (amarelo), confirmado (azul), concluído (verde), cancelado (vermelho/cinza)

## Dev Notes

### Status workflow
```
pendente → confirmado (proprietário confirma)
confirmado → concluido (atendimento realizado)
pendente|confirmado → cancelado (cliente cancela)
```

### FullCalendar Resource View
- Plugin `@fullcalendar/resource-timegrid` permite vista lado a lado
- Cada profissional é um "resource"
- Precisa instalar: `@fullcalendar/resource`, `@fullcalendar/resource-timegrid`

### Supabase Realtime
```typescript
const channel = supabase
  .channel('agendamentos-realtime')
  .on('postgres_changes', {
    event: '*',
    schema: 'public',
    table: 'agendamentos',
    filter: `salao_id=eq.${salaoId}`
  }, (payload) => { /* update state */ })
  .subscribe()
```

### Padrões
- Server Action: `updateAgendamentoStatus`
- Client Component com estado local + Realtime sync
- Filtro de profissionais como chips/checkboxes acima do calendário
