# Story 1.8: i18n — Tradução das Páginas de Dashboard (Waves 2-8)

**Status:** Ready
**Epic:** EPIC-1 Bellus — Wave 1 Fundação
**Criado por:** River (@sm)
**Data:** 2026-03-17
**Depende de:** Story 1.5 (sistema i18n base configurado)

---

## Story

**As a** usuário do Bellus que usa o sistema em PT, EN ou RU,
**I want** que todas as páginas do dashboard estejam traduzidas no meu idioma,
**so that** a experiência seja consistente em todo o sistema, não apenas nas páginas de auth e navegação.

---

## Acceptance Criteria

1. Todas as strings hardcoded nas páginas de Agenda traduzidas nos 4 idiomas (PT/ES/EN/RU)
2. Todas as strings hardcoded nas páginas de Clientes traduzidas nos 4 idiomas
3. Todas as strings hardcoded nas páginas de Serviços traduzidas nos 4 idiomas
4. Todas as strings hardcoded nas páginas de Caixa traduzidas nos 4 idiomas
5. Todas as strings hardcoded nas páginas de Fiscal traduzidas nos 4 idiomas
6. Todas as strings hardcoded na página de Configurações traduzidas nos 4 idiomas
7. Nenhuma string user-facing hardcoded em espanhol nos componentes do dashboard
8. Script `npm run i18n:check` passa com 100% coverage em todos os locales
9. Build passa sem erros (`npm run build`)

### Escopo

12 componentes com 200+ strings hardcoded em espanhol. Infraestrutura i18n (next-intl) já está configurada desde Story 1.5.

### Arquivos a modificar

| Fiscal | `fiscal-view.tsx` | ~40 |
| Config | `configuracoes/page.tsx` | ~27 |

### Padrão de migração

```typescript
// Antes
<h1>Clientes</h1>

// Depois
import { useTranslations } from "next-intl";
const t = useTranslations("clients");
<h1>{t("title")}</h1>
```

### Locale files

- `src/locales/es/common.json` — base (novas keys em ES)
- `src/locales/pt/common.json` — tradução PT-BR
- `src/locales/en/common.json` — tradução EN
- `src/locales/ru/common.json` — tradução RU (Cirílico)
