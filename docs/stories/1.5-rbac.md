## User Story

**Como** sistema,
**Quero** ter controle de acesso por roles (proprietário, profissional, cliente),
**Para que** cada tipo de usuário veja e acesse apenas o que lhe é permitido.

## Acceptance Criteria

- [x] AC-01: Role `proprietario` tem acesso completo ao dashboard
- [x] AC-02: Role `profissional` só acessa: sua própria agenda, seus agendamentos
- [x] AC-03: Role `cliente` não tem acesso ao dashboard (apenas portal público)
- [x] AC-04: Tentativa de acesso não autorizado retorna 403 ou redireciona
- [x] AC-05: Role do usuário carregado corretamente no contexto da aplicação
- [x] AC-06: Componentes UI ocultam ações não permitidas por role

## Tarefas Técnicas

- [x] T-01: Hook `useRole()` — retorna role, isProprietario, isProfissional, canAccess
- [x] T-02: Componente `<RoleGuard role="proprietario">` — protege UI por role
- [x] T-03: Verificação de role via canAccessRoute (roles.ts)
- [x] T-04: RLS policies multi-tenant (migration 009)
- [x] T-05: Auth store (Zustand) com role global
