## User Story

**Como** sistema,
**Quero** ter controle de acesso por roles (proprietário, profissional, cliente),
**Para que** cada tipo de usuário veja e acesse apenas o que lhe é permitido.

## Contexto

3 roles no sistema: proprietário (acesso total), profissional (só vê sua agenda), cliente (acesso via portal público). As RLS policies do Supabase reforçam o isolamento por salão.

## Acceptance Criteria

- [ ] AC-01: Role `proprietario` tem acesso completo ao dashboard
- [ ] AC-02: Role `profissional` só acessa: sua própria agenda, seus agendamentos
- [ ] AC-03: Role `cliente` não tem acesso ao dashboard (apenas portal público)
- [ ] AC-04: Tentativa de acesso não autorizado retorna 403 ou redireciona
- [ ] AC-05: Role do usuário carregado corretamente no contexto da aplicação
- [ ] AC-06: Componentes UI ocultam ações não permitidas por role

## Tarefas Técnicas

- [ ] T-01: Criar hook `useRole()` que retorna o role do usuário atual
- [ ] T-02: Criar componente `<RoleGuard role="proprietario">` para proteger UI
- [ ] T-03: Adicionar verificação de role no middleware Next.js
- [ ] T-04: RLS policies diferenciadas por role no Supabase
- [ ] T-05: Contexto de auth com role disponível globalmente
