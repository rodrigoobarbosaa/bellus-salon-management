# Story 9.18: Testes E2E do Fluxo Completo Verifactu

**Status:** InProgress

## Story

**As a** equipa de desenvolvimento do Bellus,
**I want** testes end-to-end que validem todo o fluxo de facturação Verifactu,
**so that** tenhamos confiança de que o sistema está conforme antes de comercializar.

---

## Acceptance Criteria

1. Teste E2E: Pagamento → Fatura gerada → Hash calculado → XML gerado → QR gerado → Estado pendiente
2. Teste E2E: Fatura enviada à AEAT (mock) → Estado aceptado → Evento registrado
3. Teste E2E: Fatura rejeitada → Retry → Sucesso no retry → Estado aceptado
4. Teste E2E: Anulação → Retificativa criada → XML anulação → Envio AEAT
5. Teste unitário: Hash encadeado — 3 faturas → cadeia íntegra → verificação OK
6. Teste unitário: Hash encadeado — fatura adulterada → verificação FALHA
7. Teste unitário: Numeração atómica — 10 faturas concorrentes → números sequenciais sem saltos
8. Teste unitário: XML válido contra schema (snapshot test)
9. Teste unitário: QR data formato correto
10. Todos os testes passam em CI (`npm test`)
11. `npm run build` sem erros
