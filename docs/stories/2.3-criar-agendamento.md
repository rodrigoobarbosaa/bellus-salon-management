## Story

Como **proprietário ou profissional**, quero **criar um agendamento** selecionando cliente, serviço, profissional e horário, para que a agenda fique organizada e sem conflitos.

---

## Acceptance Criteria

- [ ] AC-01: Modal de criação abre ao clicar slot vazio no calendário ou botão "Nuevo turno"
- [ ] AC-02: Formulário contém: cliente (busca/autocomplete), serviço (select), profissional (select), data/hora início
- [ ] AC-03: Data/hora fim calcula automaticamente baseado na duração do serviço selecionado
- [ ] AC-04: Sistema detecta conflito de horário (mesmo profissional, horário sobreposto) e bloqueia

- [ ] AC-07: Status inicial: "pendente"
- [ ] AC-08: Após criar, agendamento aparece imediatamente no calendário

## Dev Notes

### Conflito de horário
Query para validar:
```sql
SELECT id FROM agendamentos
WHERE profissional_id = $1
  AND status NOT IN ('cancelado')
  AND data_hora_inicio < $new_fim
  AND data_hora_fim > $new_inicio
```

### Cálculo de fim
`data_hora_fim = data_hora_inicio + servicos.duracao_minutos`

### Preço
```
preco_efetivo = servicos_profissionais.preco_override ?? servicos.preco_base
```

### Padrões
- Modal/Dialog usando componente `Dialog` do projeto
- Server Action para criar agendamento com validação server-side
- Busca de clientes com debounce (Client Component)
- Formato de hora: 24h (Europa)
