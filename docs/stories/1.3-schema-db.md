## User Story

**Como** sistema,
**Quero** ter as tabelas base criadas no Supabase com RLS ativo,
**Para que** todos os dados do salão sejam isolados por tenant e seguros.

## Contexto

Schema base que suporta todo o Bellus. Deve ser multi-tenant ready (cada salão é um registro na tabela `saloes`) e ter RLS (Row Level Security) ativo desde o início.

## Acceptance Criteria

- [x] AC-01: Tabela `saloes` criada com campos: id, nome, slug, criado_em
- [x] AC-02: Tabela `usuarios` criada linkada ao Supabase Auth (auth.users)
- [x] AC-03: Tabela `profissionais` criada com FK para saloes e usuarios
- [x] AC-04: RLS ativo em todas as tabelas — usuário só vê dados do seu salão
- [x] AC-05: Migrations versionadas em `bellus/supabase/migrations/`
- [x] AC-06: Tipos TypeScript gerados a partir do schema (`supabase gen types`)
- [x] AC-07: Índices criados nas colunas de FK e busca frequente

## Tarefas Técnicas

- [x] T-01: Criar migration `009_multi_tenant.sql`
- [x] T-02: Tabela `saloes` (id uuid PK, nome text, slug text unique, whatsapp text, timezone text, moeda text, criado_em timestamptz)
- [x] T-03: Tabela `usuarios` (id uuid PK references auth.users, salao_id uuid FK, role text CHECK IN ['proprietario','profissional','cliente'], nome text, email text)

## Notas Técnicas

- Usar UUIDs como PKs (uuid_generate_v4())
- RLS policies: `USING (salao_id = auth.jwt() ->> 'salao_id')` ou via função helper
- O `salao_id` do usuário deve estar no JWT claims (via custom claims Supabase)
- Trigger `on_auth_user_created` cria registro em `usuarios` automaticamente
