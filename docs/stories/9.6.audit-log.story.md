# Story 9.6: Registro de Eventos Inalterável (Audit Log SIF)

**Status:** InProgress
**Epic:** EPIC-09 Verifactu
**Criado por:** River (@sm)
**Data:** 2026-03-30
**Depende de:** Story 9.1 (schema factura_eventos)
**Complexidade:** M (5 SP)

---

## Story

**As a** sistema certificado SIF,
**I want** registrar cada evento do módulo de facturação num log inalterável,
**so that** toda operação seja rastreável e auditável conforme RD 1007/2023 (trazabilidade).

---

## Acceptance Criteria

1. Existe server action `logFacturaEvento(params)` que insere em `factura_eventos` com: tipo_evento, factura_id (nullable), salao_id, detalle (JSONB), usuario_id
2. Eventos registrados automaticamente para: emissão de fatura, anulação, retificação, envio à AEAT, erro de envio, consulta de estado
3. O campo `detalle` (JSONB) contém metadados relevantes: IP do request (quando aplicável), user-agent, valores da fatura, motivo (para anulação)
4. Tabela `factura_eventos` não aceita UPDATE ou DELETE (RLS policy)
5. Existe server action `getFacturaEventos(facturaId)` que retorna timeline de eventos ordenada por created_at
6. Existe server action `getAuditLog(salaoId, dateRange, tipoEvento?)` para consulta geral do audit log com paginação
7. Cada chamada a `createFactura` e `anularFactura` registra automaticamente o evento correspondente (já parcialmente implementado na Story 9.3 — aqui formalizamos e expandimos)
8. Eventos incluem timestamp com timezone (TIMESTAMPTZ) — nunca timestamp local
9. `npm run build` sem erros

---

## Scope

**IN:**
- Server action `logFacturaEvento`
- Registro automático em createFactura/anularFactura
- Consulta de eventos por fatura e por salão
- Garantia de imutabilidade (RLS)

**OUT:**
- Visualização UI do audit log (Story 9.14)
- Envio à AEAT (Story 9.10)
