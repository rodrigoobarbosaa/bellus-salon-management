# Story 2.2 — Calendário Visual Semanal/Diário

**Epic:** EPIC-02 Agenda & Serviços
**Status:** Done
**Prioridade:** MUST HAVE
**Depende de:** Story 2.1 (serviços existem)
**PRD Refs:** FR-01.1, FR-01.4

---

## Story

Como **proprietário ou profissional**, quero **ver a agenda semanal e diária** com os agendamentos de cada profissional, para que possa organizar o dia e evitar conflitos.

---

## Acceptance Criteria

- [x] AC-01: Calendário exibe vista semanal (padrão) e diária (toggle)
- [x] AC-02: Cada profissional tem cor distinta no calendário (campo `cor_agenda`)
- [x] AC-03: Agendamentos exibem: hora, nome do cliente, serviço, profissional
- [x] AC-04: Slots ocupados são visualmente distintos de slots livres
- [x] AC-05: Navegação entre semanas/dias com botões anterior/próximo/hoje
- [x] AC-06: Vista responsiva: desktop mostra semana, mobile mostra dia
- [x] AC-07: Clicar num slot vazio abre modal de criar agendamento (Story 2.3)
- [x] AC-08: Clicar num agendamento abre detalhes/edição (Story 2.4)

## Dev Notes

### Biblioteca de calendário
- Usar **@fullcalendar/react** (recomendação do PRD)
  - `@fullcalendar/core`, `@fullcalendar/react`, `@fullcalendar/daygrid`, `@fullcalendar/timegrid`, `@fullcalendar/interaction`
- Alternativa: react-big-calendar (mais simples mas menos features)

### Configurações do calendário
- Fuso horário: `Europe/Madrid` (do campo `saloes.timezone`)
- Horário de trabalho padrão: 09:00-20:00 (configurável futuramente)
- Slot duration: 30 minutos
- Idioma: espanhol (`es`)

### Dados necessários
- Profissionais do salão (para cores e filtro)
- Agendamentos da semana visível (query por range de datas)
- Serviços (para nome no evento)
- Clientes (para nome no evento)

### Padrões
- Client Component (calendário é interativo)
- Fetch dados via Server Component wrapper + props ou hooks
- Supabase Realtime para updates (AC do EPIC-02)
