## Story

Como **cliente**, quero **receber um lembrete 24h antes do meu agendamento**, para que não esqueça da minha reserva.

---

## Acceptance Criteria

- [ ] AC-01: API Route `/api/cron/reminders` processa lembretes pendentes
- [ ] AC-02: Busca agendamentos confirmados/pendentes nas próximas 24-25h que ainda não receberam lembrete
- [ ] AC-03: Envia lembrete via WhatsApp/SMS usando template de lembrete
- [ ] AC-04: Registra envio em notificacoes_log com tipo "lembrete_24h"
- [ ] AC-05: Agendamentos cancelados são ignorados
- [ ] AC-06: Endpoint protegido com CRON_SECRET para evitar abuso

---

## Dev Notes

- Vercel Cron ou chamada manual via cURL
- Route handler: `src/app/api/cron/reminders/route.ts`
- Usar service_role para queries (sem auth context)
- Verificar se lembrete já foi enviado (evitar duplicatas)
- Window de 24-25h para compensar frequência do cron
