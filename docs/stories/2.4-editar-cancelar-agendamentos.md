# Story 2.4 — Editar e Cancelar Agendamentos

**Epic:** EPIC-02 Agenda & Serviços
**Status:** Ready for Dev
**Prioridade:** MUST HAVE
**Depende de:** Story 2.3 (criar agendamento)
**PRD Refs:** FR-01.2, FR-01.5

---

## Story

Como **proprietário ou profissional**, quero **editar ou cancelar agendamentos existentes**, para que possa reagendar clientes e manter a agenda atualizada.

---

## Acceptance Criteria

- [ ] AC-01: Clicar num agendamento no calendário abre modal com detalhes
- [ ] AC-02: Proprietário pode editar: data/hora, serviço, profissional, notas
- [ ] AC-03: Edição revalida conflito de horário (mesma lógica de criação)
- [ ] AC-04: Proprietário pode cancelar agendamento (status → "cancelado")
- [ ] AC-05: Agendamento cancelado aparece diferente no calendário (cor/opacidade)
- [ ] AC-06: Profissional pode editar apenas seus próprios agendamentos
- [ ] AC-07: Confirmação antes de cancelar ("¿Seguro que quieres cancelar este turno?")
- [ ] AC-08: Drag-and-drop para mover agendamento no calendário (FullCalendar interaction)

---

## Dev Notes

### Status transitions
- `pendente` → `confirmado` | `cancelado`
- `confirmado` → `concluido` | `cancelado`
- `concluido` → (terminal)
- `cancelado` → (terminal)

### Drag-and-drop
FullCalendar `@fullcalendar/interaction` plugin permite:
- `eventDrop` — mover evento para outro dia/hora
- `eventResize` — alterar duração

Ambos devem revalidar conflito antes de salvar.

### Padrões
- Server Actions: `updateAgendamento`, `cancelAgendamento`
- Reutilizar componente de formulário da Story 2.3 (modo edição)
- Confirmação de cancelamento com Dialog
