# Story 10.1: Validacao XML contra XSD oficial da AEAT

**Status:** InProgress
**Epic:** EPIC-10 Certificacao AEAT
**Criado por:** Orion (@aios-master)
**Data:** 2026-03-31
**Depende de:** EPIC-09 (Verifactu completo)
**Complexidade:** M (5 SP)

---

## Story

**As a** sistema Bellus certificado SIF,
**I want** validar todo XML gerado contra o schema oficial da AEAT,
**so that** nenhum XML invalido seja enviado a AEAT, evitando rejeicoes e multas.

---

## Acceptance Criteria

1. Schema XSD da AEAT armazenado localmente em `src/lib/verifactu/schemas/`
2. Funcao `validateVerifactuXml(xml)` retorna `{ valid: boolean; errors: string[] }`
3. Validacao chamada automaticamente antes de qualquer envio a AEAT
4. XML invalido bloqueia envio e registra erro no `factura_eventos`
5. Testes unitarios: XML valido passa, XML com campos em falta falha
6. Testes snapshot: XML gerado pelo builder passa validacao
7. `npm run build` sem erros, testes passam

## Notas Tecnicas

- XSD validation com libxmljs2 tem dependencias nativas problematicas no Vercel serverless
- Abordagem: validador TypeScript estrutural que verifica todos os campos, formatos e regras da Orden HAC/1177/2024
- Mais robusto e portavel que XSD validation nativa
