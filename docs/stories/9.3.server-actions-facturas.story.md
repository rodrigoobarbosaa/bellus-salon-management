# Story 9.3: Server Actions CRUD para Geração de Faturas

**Status:** InProgress
**Epic:** EPIC-09 Verifactu

---

## Story

**As a** sistema Bellus,
**I want** server actions que criem faturas a partir de transações existentes,
**so that** cada pagamento concluído gere uma fatura fiscal completa com todos os dados necessários.

---

## Acceptance Criteria

1. Existe server action `createFactura(transacaoId: string)` que gera uma fatura a partir de uma transação concluída
2. A fatura é criada com: dados do salão (NIF, nombre_fiscal de `configuracoes_fiscais`), dados do cliente, dados do serviço, valores com IVA/IRPF calculados
3. As linhas da fatura (`factura_lineas`) são criadas para cada serviço incluído na transação (incluindo serviços adicionais de split)
4. O número é atribuído atomicamente via `get_next_factura_numero`
5. Um evento de tipo `emision` é registrado em `factura_eventos` com detalhes da operação
6. O `estado_aeat` inicial é `pendiente`
7. Existe server action `getFacturas(salaoId, filters?)` com paginação e filtros por: data, estado_aeat, cliente
8. Existe server action `getFacturaById(facturaId)` que retorna fatura completa com linhas e eventos
9. Existe server action `anularFactura(facturaId, motivo)` que: NÃO deleta a fatura original, cria uma fatura retificativa com valores negativos vinculada à original, registra evento de tipo `anulacion`
10. Validação: não é possível criar fatura se `configuracoes_fiscais` não tem NIF preenchido
11. `npm run build` sem erros

---

## Scope

**IN:**
- Server actions: createFactura, getFacturas, getFacturaById, anularFactura
- Cálculo de IVA/IRPF baseado em configuracoes_fiscais
- Criação de linhas (factura_lineas)
- Registro de eventos (factura_eventos)
- Validações de negócio

**OUT:**
- Hash encadeado (Story 9.4)
- Geração XML (Story 9.7)
- Geração PDF (Story 9.9)
- Envio AEAT (Story 9.10)
- UI (Story 9.13)
