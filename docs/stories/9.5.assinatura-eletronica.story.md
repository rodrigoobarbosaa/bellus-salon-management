# Story 9.5: Assinatura Eletrónica Qualificada

**Status:** InProgress
**Epic:** EPIC-09 Verifactu
**Criado por:** River (@sm)
**Data:** 2026-03-30
**Depende de:** Story 9.4 (hash encadeado)
**Complexidade:** L (8 SP)

---

## Story

**As a** proprietário de salão com certificado digital,
**I want** que as faturas sejam assinadas eletronicamente com meu certificado,
**so that** cumpram o requisito de assinatura qualificada do RD 1007/2023 para Verifactu.

---

## Acceptance Criteria

1. O proprietário pode fazer upload do certificado digital (.p12 / .pfx) na página de configurações fiscais
2. O certificado é armazenado de forma segura (encriptado at-rest) — NUNCA em plaintext
3. Ao criar uma fatura, o hash é assinado com o certificado digital do salão
4. O campo `firma_digital` da fatura contém a assinatura em formato Base64
5. Existe função `signFacturaHash(hash, certificado, password)` no server-side
6. Existe função `verifyFacturaSignature(hash, firma_digital, certificado_publico)` para validação
7. Se o certificado não está configurado, a fatura é criada com `firma_digital = null` e um warning é exibido (sistema funciona em modo degradado)
8. Alerta no dashboard quando o certificado expira em menos de 30 dias
9. O password do certificado é solicitado no momento do upload e usado para extrair a chave privada — NÃO é armazenado permanentemente
10. **Decisão arquitetural:** Se a complexidade de assinatura local for excessiva, integrar com API de terceiros (fiskaly/efsta/Invopop) como alternativa — documentar decisão no ADR
11. `npm run build` sem erros

---

## Notas Técnicas

### Opção A: Assinatura Local (Node.js)
- Usar `node:crypto` para assinar com certificado .p12
- Chave privada extraída do .p12 com password
- Formato: PKCS#7 ou XMLDSig (conforme AEAT)

### Opção B: API de Terceiros
- fiskaly Verifactu API: gerencia assinatura + envio AEAT
- efsta: API certificada para Verifactu
- Invopop: API open-source para facturação espanhola

**Recomendação:** Avaliar custo vs. complexidade na T1. Se <€50/mês, API de terceiros é preferível para MVP.
