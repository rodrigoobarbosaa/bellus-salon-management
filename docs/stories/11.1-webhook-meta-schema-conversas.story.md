# Story 11.1: Webhook Meta Cloud API + Schema de Conversas

**Status:** Ready
**Epic:** EPIC-11 Automação Conversacional — Wave 11
**Criado por:** Morgan (@pm)
**Data:** 2026-04-20

---

## Story

**As a** proprietária do salão,
**I want** que mensagens dos clientes no WhatsApp e Instagram cheguem ao sistema Bellus automaticamente,
**so that** o sistema possa processá-las e responder sem eu precisar estar no telemóvel.

---

## Acceptance Criteria

1. Endpoint `/api/webhooks/meta` responde ao challenge de verificação da Meta (GET)
2. Endpoint `/api/webhooks/meta` recebe e processa mensagens (POST)
3. Mensagens WhatsApp são persistidas na tabela `conversas_mensagens`
4. Mensagens Instagram DM são persistidas na mesma tabela com `canal = 'instagram'`
5. Webhook valida signature HMAC (X-Hub-Signature-256) para segurança
6. Status de entrega (delivered, read) são atualizados no log
7. Tabela `conversas` mantém sessão por cliente (última mensagem, estado)
8. Migração Supabase criada com todas as tabelas necessárias

## Dev Notes

### Schema de Conversas

```sql
-- Sessão de conversa por cliente + canal
CREATE TABLE conversas (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  salao_id UUID NOT NULL REFERENCES saloes(id),
  cliente_id UUID REFERENCES clientes(id),
  canal TEXT NOT NULL CHECK (canal IN ('whatsapp', 'instagram')),
  -- Identificador externo (telefone para WA, instagram_scoped_id para IG)
  external_id TEXT NOT NULL,
  estado TEXT NOT NULL DEFAULT 'ativo' CHECK (estado IN ('ativo', 'aguardando_humano', 'encerrado')),
  contexto JSONB DEFAULT '{}',
  ultima_mensagem_em TIMESTAMPTZ,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now(),
  UNIQUE(salao_id, canal, external_id)
);

-- Mensagens individuais
CREATE TABLE conversas_mensagens (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  conversa_id UUID NOT NULL REFERENCES conversas(id),
  salao_id UUID NOT NULL REFERENCES saloes(id),
  direcao TEXT NOT NULL CHECK (direcao IN ('recebida', 'enviada')),
  tipo TEXT NOT NULL DEFAULT 'texto' CHECK (tipo IN ('texto', 'imagem', 'audio', 'template', 'interativo')),
  conteudo TEXT,
  meta_message_id TEXT,
  status TEXT DEFAULT 'recebida' CHECK (status IN ('recebida', 'enviada', 'entregue', 'lida', 'falhou')),
  metadata JSONB DEFAULT '{}',
  created_at TIMESTAMPTZ DEFAULT now()
);
```

### Webhook Payload — WhatsApp

```json
{
  "object": "whatsapp_business_account",
  "entry": [{
    "id": "BUSINESS_ACCOUNT_ID",
    "changes": [{
      "value": {
        "messaging_product": "whatsapp",
        "metadata": { "phone_number_id": "PHONE_ID" },
        "messages": [{
          "from": "34612345678",
          "id": "wamid.xxx",
          "timestamp": "1234567890",
          "type": "text",
          "text": { "body": "Quero marcar um corte" }
        }]
      }
    }]
  }]
}
```

```json
      "recipient": { "id": "PAGE_ID" },
      "timestamp": 1234567890,
      "message": {
        "mid": "msg_id",
        "text": "Olá, quero agendar"
      }
    }]
  }]
}
```

### HMAC Validation

```typescript
import { createHmac } from "crypto";

function verifyWebhookSignature(body: string, signature: string): boolean {
  const expected = createHmac("sha256", process.env.META_APP_SECRET!)
    .update(body)
    .digest("hex");
  return `sha256=${expected}` === signature;
}
```

### Identificação de Cliente

1. WhatsApp: match por `telefone` na tabela `clientes`
2. Instagram: match por `instagram_id` (novo campo) ou criar conversa sem cliente vinculado
3. Se cliente não existe: conversa criada com `cliente_id = NULL`, vinculado depois quando fornecer dados
